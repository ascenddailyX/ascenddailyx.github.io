# Ascend Daily — Newsletter Site

Your complete newsletter website. Two files, live on the internet in under 10 minutes.

## Files

- `index.html` — Homepage with subscribe form and issue archive
- `newsletter.html` — All 3 issues in one page with switcher

## How to Publish on GitHub Pages (Step by Step)

### Step 1 — Create a GitHub account

Go to github.com → Sign Up → use your email → pick a username (e.g. `ascenddaily`)

### Step 2 — Create a new repository

1. Click the green **“New”** button on your GitHub dashboard
1. Repository name: `ascend-daily` (or `ascenddaily.github.io` for a cleaner URL)
1. Set it to **Public**
1. Check **“Add a README file”**
1. Click **“Create repository”**

### Step 3 — Upload your files

1. Inside your new repository, click **“Add file” → “Upload files”**
1. Drag and drop BOTH files:
- `index.html`
- `newsletter.html`
1. Scroll down, click **“Commit changes”**

### Step 4 — Enable GitHub Pages

1. Click **“Settings”** tab in your repository
1. Scroll down to **“Pages”** in the left sidebar
1. Under “Branch” select **main** → folder **/ (root)**
1. Click **Save**

### Step 5 — Your site is live!

Within 2-3 minutes your site will be live at:
`https://YOUR-USERNAME.github.io/ascend-daily/`

If you named the repo `ascenddaily.github.io` it will be at:
`https://ascenddaily.github.io`

## Adding a Custom Domain (Optional)

1. Buy a domain at Namecheap or Google Domains (e.g. ascenddaily.com)
1. In GitHub Pages settings → “Custom domain” → enter your domain
1. Follow the DNS instructions GitHub provides
1. Your site moves to ascenddaily.com within 24 hours

## Adding New Issues

1. Open `newsletter.html`
1. Copy an existing issue block and update the content
1. Add a new button in the issue switcher at the top
1. Re-upload the file to GitHub → changes go live automatically
